# 🦋 Nabi_Curriculum.md (운영 자동화 계획서)

##[운영 철학] 모든 자동화는 '격려(Encouragement)'를 전달하는 통로여야 하며, 기술적인 완결성보다 **참가자의 감정적 흐름(십우도의 단계)**을 우선시한다.

## 1. 프로그램 개요
* **명칭**: 다시 태어나는 나, 나비 (NA-BI)
* **컨셉**: 십우도와 타로, 아들러 심리학을 결합한 자아성장 집단상담
* **목표**: 생활양식 직면 → 내면 아이 치유 → 창조적 생명력 회복 → 불완전할 용기 획득

--- 

## 2. 도구 선정 및 연동 전략 (가성비 & 효율 중심)

| 분류 | 추천 도구 | 선정 이유 및 기존 도구 활용 |
| :--- | :--- | :--- |
| **입구/신청** | **Tally Forms** | 구글 폼보다 디자인이 유려하며, **무료**로 제출 직후 이메일 발송 가능 |
| **데이터 허브** | **Google Sheets** | 모든 활동 기록과 신청 데이터를 모으는 중심 저장소 |
| **개인 기록** | **Notion** | 선생님이 기존에 쓰시는 노션을 활용하여 '나비 성장 일지' 페이지 구축 |
| **분석/격려** | **Claude 3.5** | 활동지의 깊은 내용을 이해하고 선생님의 어조로 피드백 초안 작성 |
| **최종 선물** | **Canva** | **'데이터 대량 제작'** 기능으로 10회기 기록을 한 번에 PDF로 변환 |

## 3. 차시별 자동화 업무 설계 (업무 지시)

### [Phase 1: 여행의 시작 (0~1차시)]
* **0차시 (사전 안내)**: 신청 시 Tally에서 **'나비 여행 안내서(PDF)'** 자동 발송. 줌 링크 및 준비물 안내 자동화.
* **1차시 (변화의 의지)**: "한 폭의 그림이자 화가"라는 메시지를 담은 **웰컴 메시지** 발송.

### [Phase 2: 자취를 따라 (2~4차시)]
* **2~3차시 (초기기억/생활양식)**: 
    * 활동지 사진 업로드 → **구글 드라이브** 자동 저장.
    * 노션 DB에 '나의 생활양식 삼단논법' 데이터 자동 기록.
* **4차시 (치유와 짐 내려놓기)**: 
    * 고통스러운 기억을 다루는 회기인 만큼, 종료 후 **"오늘 고생 많으셨습니다"**라는 위로의 예약 문자/메일 발송.

### [Phase 3: 날개를 펴다 (5~7차시)]
* **5~6차시 (강점/타로)**: 
    * 뽑은 타로 카드의 의미와 강점 키워드를 **캔바 템플릿**에 자동 연동하여 '나의 강점 카드' 이미지 생성.
* **7차시 (행동과 신뢰)**: 
    * "남의 시선" vs "나의 시선"을 구분하는 간단한 체크리스트를 모바일로 전송하여 실시간 응답 수집.

### [Phase 4: 본연의 모습으로 (8~9차시)]
* **8차시 (길잡이 별)**: 내가 지향하는 가치(길잡이 별)를 입력하면 **노션 대시보드 메인**에 자동으로 노출되게 설정.
* **9차시 (종강)**: **[가장 중요한 자동화]** * 0~9차시까지 구글 시트에 쌓인 참가자의 모든 기록(강점, 초기기억, 타로 등)을 캔바(Canva)와 연결.
    * **"000님의 다시 태어나는 나비 성장 포트폴리오"** PDF를 1분 만에 대량 생성하여 이메일 선물.

---

## 4. 다온의 꼼꼼한 체크리스트 (가이드라인)
1.  **정보 누수 방지**: 참가자가 노션에 올린 과제가 선생님의 '마스터 데이터베이스'에만 보이고 다른 참가자에게는 보이지 않도록 권한 설정을 꼼꼼히 확인해야 합니다.
2.  **데이터의 일관성**: Tally에서 받은 성함과 이메일 주소가 구글 시트, 노션과 일치하도록 키(Key) 값을 잘 관리해야 합니다.
3.  **따뜻함 유지**: 모든 자동화 메시지는 "시스템이 보낸 것"처럼 보이지 않게, 선생님의 평소 어조(아들러의 격려)를 반영하여 작성합니다.
